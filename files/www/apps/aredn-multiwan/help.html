PollyWAN r6 for AREDN

PollyWAN is an optional package for the MikroTik hAP ac lite, hAP ac2, and hAP ac3. It regulates three local WAN candidates, integrates with AREDN Mesh WAN routing, and is disabled and inert after installation.

Inputs

Logical nameInputPrivate table
wanWAN 1: AREDN Wi-Fi client on wlan0/wlan1, or selected Ethernet br-wan.101

Remote Mesh WANBabel-learned fallback, not a local candidate.22
Wi-Fi rule: AREDN Wi-Fi client mode keeps the logical interface name wan. Wi-Fi WAN and Ethernet WAN 1 cannot be active simultaneously. PollyWAN never changes radio mode and removes WAN 1 from the Ethernet role choices while Wi-Fi owns it.

Use hAP Ethernet port roles to assign LAN, WAN 1, WAN 2, disabled, and tagged DtD VLAN 2. Applying starts a timed rollback; reconnect and select Confirm working.

Phone USB/PdaNet

Android phone -> data USB cable -> hAP USB host -> RNDIS/CDC -> wan3 DHCP
                                                        -> optional HTTP CONNECT proxy

Enter the proxy address, TCP port, and optional credentials on the hAP. Common PdaNet values are 192.168.49.1:8000. The proxy carries public IPv4 TCP, not general UDP or incoming connections.

GPS safety: when WAN 3 is disabled, PollyWAN never opens /dev/ttyACM0 or /dev/ttyUSB0, edits gpsd, changes AREDN GPS time/location, changes radio mode, or changes USB power. It searches only /sys/class/net after WAN 3 is explicitly enabled.

Adaptive selection

Reachability is a hard gate. A user-selected HTTPS range object classifies each path as low (≤5 Mbps), medium (>5–30 Mbps), or fast (>30 Mbps). Better links require promotion hysteresis; hard failures demote immediately.

Mesh WAN and Babel

Table 26 is the selected local default, table 27 its connected subnet, and table 28 is published only when the selected path is healthy and meets the mesh-share bin. Table 22 remains the remote Mesh WAN. A protocol-boot default is denied from Babel so only PollyWAN's qualified protocol-static table-28 route is advertised.

Tunnels

WireGuard and tun* ingress may use AREDN mesh routes but is hard-blocked from local or remote Internet defaults. This policy is not a checkbox.

Recovery

uci -c /etc/config.mesh set aredn.multiwan.enabled='0'
uci -c /etc/config.mesh commit aredn
/usr/local/bin/wan-port-manager restore
/usr/local/bin/wan3-manager disable
/etc/init.d/wan3-manager stop

See /usr/share/doc/aredn-multiwan/ for complete setup, build, test, Wi-Fi WAN, GPS, and rollback instructions.
