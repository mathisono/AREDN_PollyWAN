PollyWAN setup and recovery

PollyWAN is disabled after installation. It does not publish local WAN defaults, change ports, scan USB, or alter GPS until an administrator enables it.

Android USB Tether

Android phone -> data-capable USB cable -> hAP USB host -> RNDIS/CDC/NCM -> wan3 DHCP

Connect a data-capable USB cable, unlock the Android phone, open hotspot/tethering settings, enable USB tethering, enable WAN 3 in PollyWAN, wait for DHCP, then verify health before selecting WAN 3. USB charging alone is insufficient.

The phone supplies DHCP, gateway, DNS, and NAT Internet access. Supported phones normally expose RNDIS, CDC Ethernet, or CDC NCM, but driver availability depends on the AREDN kernel and hardware. PollyWAN reports runtime USB-network support and leaves WAN 1/WAN 2 usable if WAN 3 USB support is unavailable.

GPS safety: when WAN 3 is disabled, PollyWAN never opens /dev/ttyACM0 or /dev/ttyUSB0, edits gpsd, changes AREDN GPS time/location, changes radio mode, or changes USB power. It searches only /sys/class/net after WAN 3 is explicitly enabled.

Health and Selection

Health validates the private table, source-bound gateway reachability, and direct source-bound HTTPS using curl --interface "$SOURCE" --proxy ''. Cloudflare speed tests use the same direct source-bound path. Node-to-node iperf3 tests are unchanged.

Recovery

Use the PollyWAN disable action or remove the package to restore ordinary AREDN WAN behavior. Route rollback restores main, table 26, table 27, and table 28 snapshots after failed selection.
